# VOC-AI 分析系统（达观数据）

## 系统架构
```
VOC电商平台 → 本地数据库 → AI分析引擎 → PowerBI → 业务报告
```

## 五层架构

### 1. 数据采集层
- **VOC系统**：从淘宝、京东等电商平台爬取原始评论数据
- **数据传输**：将爬取的评论数据传输到本地数据库

### 2. 数据存储层
- **本地数据库**：存储原始评论数据、处理后数据和标签数据
- **数据管理**：确保数据的安全性和完整性

### 3. AI分析层
- **数据预处理**：清洗评论文本，进行数据结构化处理
- **标签体系**：应用业务用户自定义的各品类标签体系
- **AI分析引擎**：对评论进行文本分类、情感分析和标签自动标注

### 4. 数据可视化层
- **PowerBI系统**：连接处理后的数据
- **自定义维度**：业务用户根据需求设置分析维度
- **交互式仪表盘**：生成多维度分析的可视化报表

### 5. 业务应用层
- **报告生成**：定期分析报告和专题分析报告
- **决策支持**：基于分析结果提供产品改进、营销策略调整和客户服务优化建议

---

## 炊具VOC-AI分析需求（2026年1月）

### 三大目标
1. **细化分析颗粒度**：通过AI实现更精细的用户情感、关键词及需求分析
2. **流程自动化**：将人工分析环节（数据清洗、关键词定义、情感判断）交由AI处理
3. **系统化分析框架**：构建可重复使用的分析流程，支持自定义训练或优化模型

### 核心诉求
- 人工做右上角汇总数据太耗时
- 希望对内容中的主要观点有具体量、占比的呈现
