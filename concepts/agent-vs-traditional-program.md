# Agent vs 传统程序

## 核心区别

| 维度 | 传统程序 | AI Agent |
|------|---------|----------|
| 交互方式 | 固定指令流程 | 目标驱动、自主规划 |
| 输出 | 生成文本 | **生成行动并执行行动** |
| 适应性 | 需手动调整 | 自我调整策略、重试 |
| 错误处理 | 报错即结束 | 自我纠错、迭代优化 |
| 记忆 | 仅当前输入 | 短期+长期记忆 |
| 比喻 | 自动售货机：投币→按按钮→出商品 | 私人助理：告诉需求→助理规划→完成任务 |

## 思维转变

> 从对话框问答进化为**目标驱动的任务执行**

## 五大核心特性对比

| 特性 | 传统 AI | Agent |
|------|---------|-------|
| 自主性 | 无，需逐步指导 | 独立运作 |
| 反应性 | 被动响应 | 感知环境及时响应 |
| 主动性 | 被动 | 能主动采取行动 |
| 工具使用 | 无法调用外部工具 | 可调用搜索/API/代码等 |
| 错误处理 | 输出即结束 | 可自我纠错、重试 |

## Agent 类型

| 类型 | 说明 |
|------|------|
| 单一任务 Agent | 专注特定任务，如智能客服 |
| 多模态 Agent | 处理文本、图像、语音等多种信息 |
| 自主 Agent | 较高自主性，长期运行，如自动驾驶 |
| 模拟 Agent | 在虚拟环境中进行模拟、测试和训练 |
| 反应式 Agent | 基于当前感知做出即时响应 |
| 目标导向 Agent | 围绕特定目标规划行动 |
| 实用型 Agent | 通过效用函数评估多种行动选择最优 |
| 学习型 Agent | 从经验中学习，优化决策策略 |
| 多智能体系统 | 多个 Agent 协作分工，各司其职 |

## 常见挑战与局限性

- **幻觉问题 (Hallucination)**：生成看似合理但实际错误的信息
- **边界失控 (Scope Creep)**：自主性过高导致超出预期范围的操作
- **成本控制**：多轮迭代调用 LLM 和工具成本较高
- **安全与隐私**：可能访问敏感数据
