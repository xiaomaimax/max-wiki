# MaxMES 运维手册

## 3. 运维服务级别

| 服务级别 | 可用性要求 | 响应时间 | 解决时间 | 适用场景 |
|----------|------------|----------|----------|----------|
| **金牌服务** | 99.95% | <15分钟 | <4小时 | 核心业务系统 |
| **银牌服务** | 99.9% | <30分钟 | <24小时 | 重要业务系统 |
| **铜牌服务** | 99.8% | <1小时 | <3天 | 一般业务系统 |

本MES系统采用**金牌服务**级别。

## 4. 运维团队职责

### 4.2 运维团队职责

| 角色 | 主要职责 |
|------|----------|
| **运维总监** | 负责运维团队管理和整体运维策略制定 |
| **系统运维组** | 负责服务器、网络设备、存储设备的管理和维护 |
| **应用运维组** | 负责MES应用程序、中间件的管理和维护 |
| **数据库运维组** | 负责数据库的管理、维护和优化 |
| **安全运维组** | 负责系统安全、数据备份和恢复 |
| **业务运维组** | 负责业务支持、用户培训和问题处理 |

### 4.3 具体岗位职责

| 角色 | 职责 |
|------|------|
| **服务器管理员** | 服务器硬件和操作系统的安装、配置和维护；监控服务器性能；处理服务器硬件故障 |
| **网络管理员** | 网络设备的配置和维护；监控网络性能；处理网络故障和安全事件 |
| **应用管理员** | MES应用程序的部署和配置；监控应用程序运行状态；处理应用程序故障和性能问题 |
| **中间件管理员** | 中间件（Tomcat/Redis/RabbitMQ）的配置和维护；监控中间件性能 |
| **数据库管理员** | 数据库的安装、配置和维护；监控数据库性能；执行数据库备份和恢复；数据库优化和索引管理 |
| **安全管理员** | 系统安全策略制定和实施；监控安全事件；处理安全漏洞和入侵事件；用户权限管理 |
| **备份管理员** | 制定和实施数据备份策略；执行数据备份和恢复操作；验证备份数据的完整性 |
| **业务支持工程师** | 业务用户的培训和支持；处理业务用户的问题和需求 |

## 5. 监控体系

### 5.2 监控范围

| 监控对象 | 监控内容 | 监控工具 | 负责人 |
|----------|----------|----------|--------|
| **服务器** | CPU/内存/磁盘使用率、磁盘I/O、网络流量 | Zabbix、Prometheus | 服务器管理员 |
| **网络设备** | 网络带宽、延迟、丢包率、设备状态 | Zabbix、Nagios | 网络管理员 |
| **数据库** | 连接数、查询响应时间、缓存命中率、表空间使用率 | Oracle EM、SQL Server SSMS | 数据库管理员 |
| **中间件** | 线程数、连接池、响应时间、错误率 | JConsole、WebLogic Console | 中间件管理员 |
| **应用程序** | 响应时间、错误率、并发用户数、事务成功率 | AppDynamics、New Relic | 应用管理员 |
| **系统日志** | 错误日志、警告日志、安全日志 | ELK Stack、Splunk | 安全管理员 |

### 5.3 监控工具

| 工具名称 | 用途 | 部署方式 |
|----------|------|----------|
| **Zabbix** | 服务器和网络设备监控 | 集中式部署 |
| **Prometheus + Grafana** | 系统性能监控和可视化 | 分布式部署 |
| **ELK Stack** | 日志收集和分析 | 分布式部署 |
| **AppDynamics** | 应用性能监控 | SaaS部署 |

### 5.4 监控阈值

| 监控指标 | 警告阈值 | 严重阈值 | 处理方式 |
|----------|----------|----------|----------|
| CPU使用率 | >80% | >95% | 分析进程占用，优化资源分配 |
| 内存使用率 | >80% | >90% | 分析内存泄漏，增加内存或优化应用 |
| 磁盘使用率 | >80% | >90% | 清理磁盘空间，扩展磁盘容量 |
| 磁盘I/O | >80MB/s | >100MB/s | 优化磁盘配置 |
| 网络流量 | >80%带宽 | >90%带宽 | 分析流量来源，优化网络配置 |
| 数据库连接数 | >80%最大连接数 | >90%最大连接数 | 优化连接池配置，增加最大连接数 |
| 应用响应时间 | >2秒 | >5秒 | 分析应用性能瓶颈，优化代码 |

## 6. 故障管理

### 6.3 故障级别

| 故障级别 | 定义 | 响应时间 | 解决时间 | 上报流程 |
|----------|------|----------|----------|----------|
| **P0（紧急）** | 系统完全无法使用，影响核心业务，导致业务中断 | <15分钟 | <4小时 | 立即上报运维总监、技术总监、项目经理 |
| **P1（高）** | 系统部分功能无法使用，影响主要业务，导致业务受限 | <30分钟 | <24小时 | 2小时内上报运维总监、技术总监 |
| **P2（中）** | 系统功能受限，影响次要业务，不影响核心业务 | <1小时 | <3天 | 4小时内上报运维组长 |
| **P3（低）** | 系统功能正常，存在轻微问题，不影响业务运行 | <2小时 | <7天 | 24小时内上报运维组长 |

### 7.2 性能监控指标

| 系统层级 | 监控指标 | 目标值 |
|----------|----------|--------|
| **应用层** | 响应时间、并发用户数、事务成功率、错误率 | 响应时间<2秒，成功率>99.9% |
| **中间件层** | 线程数、连接池利用率、缓存命中率 | 连接池利用率<80%，缓存命中率>90% |
| **数据库层** | 查询响应时间、连接数、缓存命中率、表空间使用率 | 查询响应时间<1秒，表空间使用率<80% |
| **服务器层** | CPU使用率、内存使用率、磁盘使用率、磁盘I/O | CPU<80%，内存<85% |
| **网络层** | 网络带宽、延迟、丢包率 | 延迟<50ms，丢包率<0.1% |

## 8. 备份与恢复

### 8.1 备份策略

| 备份类型 | 备份内容 | 备份频率 | 备份方式 | 保留期限 | 负责人 |
|----------|----------|----------|----------|----------|--------|
| **全量备份** | 完整的数据库和应用数据 | 每周日 | 离线备份 | 30天 | 备份管理员 |
| **增量备份** | 自上次全量备份以来的变更数据 | 每天 | 在线备份 | 7天 | 备份管理员 |
| **日志备份** | 数据库事务日志 | 每小时 | 在线备份 | 14天 | 备份管理员 |
| **配置备份** | 系统配置文件、应用配置文件 | 每周 | 在线备份 | 60天 | 系统管理员 |
| **代码备份** | 应用程序代码 | 每次代码变更 | 版本控制 | 永久 | 开发人员 |
